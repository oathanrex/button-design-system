Button Hierarchy

Primary, secondary, and tertiary buttons establish visual hierarchy for actions. Use primary for the main action, secondary for supporting actions, and tertiary for low-emphasis actions.

Primary Actions .btn--primary

Secondary Actions .btn--secondary

Tertiary Actions .btn--ghost

Semantic Variants

Color variants communicate intent and provide visual feedback. All colors meet WCAG 2.1 AA contrast requirements.

Solid Style

Outline Style .btn--outline

Ghost Style .btn--ghost

Destructive Actions

Use danger variants for irreversible or destructive actions that require user attention. Consider adding a confirmation dialog for critical actions.

Sizes

Size modifiers adjust button dimensions for different contexts and use cases.

Small buttons (32px) are below the 44px WCAG AAA touch target recommendation. On mobile devices, buttons are automatically enlarged to meet touch target requirements.

Shapes

Shape modifiers alter border-radius for different visual treatments.

Icon Buttons

Icons enhance button meaning and can improve scannability.

Icon-only buttons must have an aria-label attribute describing the action. All decorative icons use aria-hidden="true" and focusable="false".

Button States

Buttons communicate their state through visual changes. Hover, tab, and click to see different states in action.

Enabled vs Disabled

Loading State

Click to trigger loading state. The button will auto-reset after the specified duration. Screen readers are notified of state changes.

Toggle / Selected State

Toggle buttons maintain their pressed state using aria-pressed.

Toggle Group (Radio-like)

Only one button in the group can be selected at a time. Navigate with arrow keys.

Call-to-Action

High-emphasis buttons for primary user journeys and conversions. Use sparingly - one CTA per screen is ideal.

Floating Action Buttons

FABs represent the primary action on a screen. Typically positioned fixed in the bottom-right corner.

FAB Color Variants

Badge Buttons

Buttons with notification counts or indicators. The badge count is announced to screen readers via aria-describedby.

Dropdown Buttons

Buttons that reveal additional options in a menu.

Full keyboard navigation: Arrow keys to navigate, Enter/Space to select, Escape to close, Tab to close and move focus.

Split Buttons

Split buttons combine a primary action with a dropdown for secondary options. The main button performs the default action immediately.

Button Groups

Related buttons grouped together with connected styling. Always include role="group" and aria-label for accessibility.

Expand/Collapse Buttons

Buttons that toggle visibility of related content. The aria-expanded and aria-controls attributes are managed automatically.

This is the expandable content area. It becomes visible when the button is clicked. The button's aria-expanded attribute is toggled automatically, and screen readers announce the state change.

Navigation Buttons

Buttons designed for navigation contexts. Use aria-current="page" to indicate the current page.

Form Action Buttons

Specialized buttons for form submission and reset actions.

Auto-Disable After Click

Prevents double-submission by disabling the button after click. The button re-enables after 3 seconds for this demo.

Full Width Buttons

Block-level buttons that span the full container width. Ideal for mobile layouts and form CTAs.

High Contrast Accessibility

Maximum contrast buttons for users with visual impairments. These meet WCAG AAA contrast requirements (7:1 ratio).

Focus States

Visible focus indicators are critical for keyboard navigation. Focus rings are shown only for keyboard interaction (:focus-visible), not for mouse clicks.

Use Tab to navigate through these buttons and observe the focus ring. The ring color and offset are customizable via design tokens.

Data Attributes

data-toggle — Makes button toggleable
data-toggle-group="name" — Radio-like toggle group
data-loading-text="..." — Text shown during loading
data-loading-duration="ms" — Loading state duration (100-30000ms)
data-auto-disable — Disables after click
data-disable-duration="ms" — Auto-disable duration (100-30000ms)

JavaScript API

buttonSystem.setLoading(button, duration) — Trigger loading state (returns Promise)
buttonSystem.cancelLoading(button) — Cancel loading state
buttonSystem.setPressed(button, pressed) — Set toggle state
buttonSystem.setGroupValue(group, button) — Set active button in group
buttonSystem.openDropdown(dropdown) — Open dropdown
buttonSystem.closeDropdown(dropdown) — Close dropdown
buttonSystem.closeAllDropdowns() — Close all dropdowns
buttonSystem.destroy() — Clean up all event listeners

Custom Events

toggle — Fired when toggle button state changes (detail: { pressed })
loadingComplete — Fired when loading state ends
select — Fired when dropdown item is selected (detail: { value, item })
expand — Fired when expand/collapse button is clicked (detail: { expanded })

ARIA Attributes

aria-pressed — Toggle state (true/false)
aria-expanded — Expand/collapse state
aria-disabled — Disabled state (interactive but non-functional)
aria-busy — Loading state
aria-haspopup — Has dropdown menu
aria-controls — ID of controlled element
aria-label — Accessible name for icon buttons
aria-describedby — Additional description (e.g., badge count)
aria-current — Current page in navigation
